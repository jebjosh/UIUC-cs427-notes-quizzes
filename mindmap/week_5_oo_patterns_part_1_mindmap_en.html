① Relationship types

How do classes connect?

  • △ Generalization — is-a inheritance; abstract class → italic name
  • — Association — the most general link
  • ◇ Aggregation — part–whole; part can exist alone
  • ◆ Composition — strong whole–part; lifetimes bound

Place the diamond on the whole side.

② Multiplicity

How to read numbers on a line?

  • Multiplicity sits on one end of the link and constrains objects at that end
  • Meaning of 0..1, 1..*, 0..*, etc.

Quiz figures (Article–Author–Book, Publication): read each end separately.

③ Class Diagram

What belongs in static structure?

  • Tell Class / Attribute / Operation apart (nouns vs verbs)
  • Structure only—not algorithms, time, or interaction flow

Example: in a restaurant spec, Order is often an operation, not a class.

④ Sequence Diagram

Interaction over time

  • Horizontal: objects and lifelines; vertical: Time ↓
  • Arrows: messages
  • Order by triggering time, not importance or duration
⑤ OO modeling

Map concepts to notation

  • Class, Object, Inheritance, Encapsulation, Association
  • Match terms to symbols in class and sequence diagrams
⑥ Exam quick tips

Common mistakes

  • Part-whole → ◇ / ◆, not △
  • Queue backed by an array → Composition
  • Class diagram = structure; sequence = behavior over time
